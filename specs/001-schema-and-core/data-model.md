# Data Model: Schema and Core (Phase 0)

The authoritative shape is `schema/adr.schema.ts` (Zod). This document summarizes
the entities the core operates on; it does not redefine the contract.

## Entity: Decision Record (`Adr`)

A parsed record. Composed of:

- `frontmatter: AdrFrontmatter` — the typed, validated metadata (see below).
- `body: string` — raw markdown below the frontmatter, preserved byte-for-byte.
- `path: string` — repo-relative file path, e.g. `docs/adr/0007-...md`.
- `log?: string` — log name for federated corpora; undefined single-repo.

### `AdrFrontmatter` (selected fields; full set in the Zod source)

| Field | Type | Notes |
|---|---|---|
| `schemaVersion` | semver string | defaults to `0.1.0` |
| `id` | `NNNN`+ or ULID | unique across the corpus (corpus invariant) |
| `title` | string 3–120 | imperative phrase |
| `status` | enum | draft \| proposed \| accepted \| rejected \| superseded \| deprecated |
| `date` | ISO date | calendar-valid |
| `deciders` / `consulted` / `informed` | Identity[] | `@handle` \| `team:slug` \| email |
| `scope` | enum | component \| domain \| org |
| `reversibility` | enum | two-way-door \| one-way-door \| unknown |
| `blastRadius` | enum | component \| team \| cross-team \| org |
| `supersedes` | AdrRef[] unique | must resolve (corpus invariant) |
| `supersededBy` | AdrRef? | required iff status = superseded |
| `relatesTo` / `conflictsWith` | AdrRef[] | must resolve; conflicts → warn |
| `affects` | AffectsMatcher[] | typed matchers (resolved in Phase 1) |
| `assertions` | Assertion[] | executable guardrails (later phases) |
| `provenance` | Provenance? | authorship + `importedFrom` |
| `review` | Review? | ARB routing state |
| `evaluation` | Evaluation? | tooling-written only |
| `reviewBy` | ISO date? | expiry |

### Intra-record invariants (already in the Zod `.refine()` chain)

1. `status = superseded` ⇒ `supersededBy` present, and its converse.
2. `status = accepted` ⇒ ≥1 `deciders`, unless `provenance.importedFrom` present.
3. `provenance.authoredBy = agent` & `status = accepted` ⇒ `provenance.ratifiedBy`.
4. `reversibility = one-way-door` ⇒ `review.tier ≠ auto`.
5. Strict object: unknown frontmatter keys are rejected.

## Entity: Corpus

The set of `Adr` discovered under `docs/adr/` (excluding `0000-template.md`),
sorted by path. Provides lookup by `id` and the reference set for resolution.

### Corpus invariants (checked across records)

- **Unique ids**: no two records share an `id` (report both paths).
- **Reference resolution**: every `supersedes`, `supersededBy`, `relatesTo`,
  `conflictsWith` target resolves to a record in the corpus (same-log). Unresolved
  required references are `error`.

## Entity: Finding

A validation result.

| Field | Type | Notes |
|---|---|---|
| `rule` | string | stable rule id, e.g. `superseded-requires-supersededBy` |
| `severity` | enum | error \| warn \| info |
| `message` | string | human-readable |
| `path` | string? | record file path |
| `id` | string? | record id where known |
| `field` | string? | offending frontmatter path where applicable |

Rendering: human (grouped by file, one line per finding) and `--json` (array of
findings). Exit code is non-zero iff any `error` finding exists.

## Entity: Published Schema Artifact

`schema/adr.schema.json` — generated from the Zod source via `z.toJSONSchema`.
Not hand-edited; regenerated by `schema:emit`; equality with a fresh emit is the
`schema-emit-matches` gate.
