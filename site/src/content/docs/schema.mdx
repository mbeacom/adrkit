adrkit records are **typed data with a markdown body**. The typed half —
everything in the YAML frontmatter — is described by a single
[JSON Schema 2020-12](https://json-schema.org/) document: a strict superset of
[MADR](https://adr.github.io/madr/), with governance, routing, provenance, and
enforcement metadata added.

## Canonical URL

The schema is served, byte-for-byte, at the URL baked into its own `$id`:

```
https://adrkit.dev/schema/adr/v0.1.0/adr.schema.json
```

<Aside type="tip" title="The URL is the contract">
	This site serves the schema at exactly the path in its `$id`. That is what
	makes `$schema` references and JSON-Schema `$ref` resolution work from any
	editor or validator. The version segment (`v0.1.0`) tracks `SCHEMA_VERSION`
	in the source of truth (`schema/adr.schema.json`), and a build-time guard
	asserts the served bytes match the canonical file — they cannot drift.
</Aside>

## Reference it from your records

Point your tooling at the canonical URL so you get validation and completion as
you type.

<Tabs>
<TabItem label="ADR frontmatter (YAML)">

Add the [`yaml-language-server`](https://github.com/redhat-developer/yaml-language-server)
modeline as the first line inside the frontmatter block. Editors with the YAML
extension (VS Code, Neovim, …) then validate the record live:

```markdown
---
# yaml-language-server: $schema=https://adrkit.dev/schema/adr/v0.1.0/adr.schema.json
id: "0042"
title: Use server-side rendering for authenticated routes
status: accepted
date: 2026-07-18
---

# ADR-0042: Use server-side rendering for authenticated routes
```

</TabItem>
<TabItem label="Standalone JSON">

A JSON instance of the frontmatter can reference the schema directly with the
`$schema` keyword:

```json
{
  "$schema": "https://adrkit.dev/schema/adr/v0.1.0/adr.schema.json",
  "id": "0042",
  "title": "Use server-side rendering for authenticated routes",
  "status": "accepted",
  "date": "2026-07-18"
}
```

</TabItem>
<TabItem label="VS Code (workspace)">

Map the schema to your ADR files in `.vscode/settings.json` so every record in
the folder is validated without a per-file modeline:

```json
{
  "yaml.schemas": {
    "https://adrkit.dev/schema/adr/v0.1.0/adr.schema.json": "docs/adr/*.md"
  }
}
```

</TabItem>
</Tabs>

## Required fields

Every record must declare at least these four. Everything else has a schema
default or is optional.

| Field | Type | Notes |
| --- | --- | --- |
| `id` | string | `0042` (same log) or a 26-char ULID. |
| `title` | string | 3–120 characters. |
| `status` | enum | `draft`, `proposed`, `accepted`, `rejected`, `superseded`, `deprecated`. |
| `date` | date | `YYYY-MM-DD`. |

## Governance fields that do the work

These are what make a record enforceable and routable rather than advisory.

| Field | Purpose |
| --- | --- |
| `affects` | Typed matchers (`path`, `package`, `entity`, `resource`, `api`, `data`) declaring what the decision governs. This is what lets CI and agents answer *"which decisions govern this change?"* |
| `reversibility` | `two-way-door`, `one-way-door`, or `unknown`. A `one-way-door` decision may not take the auto-approve fast path. |
| `blastRadius` | `component`, `team`, `cross-team`, or `org`. |
| `supersedes` / `supersededBy` | Links the supersession graph; validated for cycles by `adr lint`. |
| `provenance` | Who (or what agent) authored the record, and who ratified it. An agent-authored record cannot reach `accepted` without a named human ratifier. |

The schema is `additionalProperties: false` — unknown frontmatter keys are a hard
error, so a typo fails validation instead of silently doing nothing.

## Versioning and stability

- The schema is generated from a Zod source of truth
  (`schema/adr.schema.ts`) and emitted to `schema/adr.schema.json`; CI fails if
  they diverge.
- The `$id` **origin and path are fixed for the life of the major version** —
  see [ADR-0011](/adr/0011-host-the-canonical-json-schema-at-its-id-on-adrkit-dev/). Additive
  changes are minor; breaking changes require a major version and a new `$id`
  path.
- The schema is released under [CC0](https://github.com/mbeacom/adrkit/blob/main/schema/LICENSE)
  so competing implementations can adopt it with no license consideration at all.
